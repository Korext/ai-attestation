# Ethics

## Principles

AI Attestation is built on the
following principles:

1. Transparency over obscurity.
   AI code authorship should be
   visible, not hidden.

2. Developer control. Developers
   choose to install attestation.
   It is not surveillance. The
   developer owns the data.

3. Tool neutrality. AI Attestation
   tracks all AI coding tools
   equally. It does not favor or
   penalize any tool.

4. Privacy by design. Attestation
   tracks commit metadata only.
   It never reads, stores, or
   transmits source code.

5. Opt-out respect. Any developer
   can remove the attestation file
   or uninstall the git hook at
   any time.

## Data Handling

Attestation data includes:
  Commit counts
  AI tool names
  Date ranges
  Governance status

Attestation data does NOT include:
  Source code
  Prompt content
  Developer identities beyond
  git commit authors (which are
  already public in git history)
