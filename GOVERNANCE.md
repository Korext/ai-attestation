# Governance

## Maintainer Model

AI Attestation is maintained by the
Korext team.

## Lead Maintainer

Tom Bruno, Founder, Korext
Contact: maintainers@korext.com

## Specification Changes

Changes to SPEC.md require:

1. Open an issue with the proposed
   change
2. 14 day comment period
3. Maintainer approval

## Tool Registry Updates

Adding a new AI tool to the
detection registry requires:

1. Evidence of the tool's commit
   patterns (co-author trailers,
   commit message patterns, or
   file headers)
2. Links to public documentation
   or repos showing the pattern
3. PR with the tool added to
   src/tools.js

## Schema Changes

Changes to schema.json require:

1. Backward compatibility with
   existing attestation files
2. Version bump in the schema
3. Updated documentation in SPEC.md
